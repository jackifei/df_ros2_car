## 各文件/模块说明

### 1. 核心算法层（必须自己实现，ROS2 没有提供）

#### [ackermann_differential.py](../src/joystick_bridge/joystick_bridge/ackermann_differential.py) — 阿克曼电子差速

| 项目 | 说明 |
|------|------|
| **作用** | 根据车辆线速度 v 和角速度 ω 计算左右后轮速度差 |
| **核心公式** | `δ = atan(L·ω/v)`, `v_left = v·(R-W/2)/R`, `v_right = v·(R+W/2)/R` |
| **关键参数** | wheelbase (轴距 L)、track_width (后轮轮距 W) |
| **ROS2自带？** | ❌ 否，是阿克曼底盘特有的应用层算法 |

#### [wheel_odometry.py](../src/joystick_bridge/joystick_bridge/wheel_odometry.py) — 轮式里程计

| 项目 | 说明 |
|------|------|
| **作用** | 基于后轮左右速度进行航位推算，发布 `/odom` + `/tf` |
| **核心公式** | `v = (vL+vR)/2`, `ω = (vR-vL)/W`, 积分步长 Δt=0.02s |
| **发布话题** | `/odom` (nav_msgs/Odometry), `/tf` (odom→base_link) |
| **ROS2自带？** | ❌ 否，robot_localization 只做融合不做轮式里程计生成 |

#### [joystick_bridge_node.py](../src/joystick_bridge/joystick_bridge/joystick_bridge_node.py) — 手柄桥接主节点

| 项目 | 说明 |
|------|------|
| **作用** | 订阅手柄 `/cmd_vel` → 调用阿克曼差速 → 发布轮速 + 更新里程计 |
| **订阅话题** | `/cmd_vel` (geometry_msgs/Twist) |
| **发布话题** | `/odom`, `/tf`, `/wheel_speeds/left`, `/wheel_speeds/right` |
| **运行频率** | 50Hz（定时器驱动） |
| **安全机制** | 看门狗：500ms无手柄数据自动停车 |
| **ROS2自带？** | ❌ 否，业务逻辑链路需要自己编排 |

### 2. 配置层（ROS2 自带 EKF 引擎，只需配置文件）

#### [ekf_params.yaml](../src/robot_localization_config/config/ekf_params.yaml) — EKF 融合参数

| 项目 | 说明 |
|------|------|
| **作用** | 配置 robot_localization 的 ekf_node 进行多传感器融合 |
| **融合输入** | `/odom` (vx + vyaw) + `/imu/data` (yaw + vyaw + ax + ay) |
| **EKF状态** | 15维 [x,y,z,roll,pitch,yaw, vx,vy,vz,vroll,vpitch,vyaw, ax,ay,az] |
| **输出** | `/odometry/filtered` (融合里程计) + `/tf` |
| **ROS2自带？** | ✅ `robot_localization` 是 ROS2 官方包，我们只提供 **YAML配置** |
| **关键配置** | frequency=50Hz, two_d_mode=true, sensor_timeout=0.06s |

#### [ekf_localization.launch.py](../src/robot_localization_config/launch/ekf_localization.launch.py) — EKF 启动文件

| 项目 | 说明 |
|------|------|
| **作用** | 启动 `robot_localization` 的 `ekf_node`，加载 ekf_params.yaml |
| **ROS2自带？** | ✅ `ekf_node` 可执行文件是 ROS2 自带的，launch 文件是我们写的 |

## 启动方式

```bash
# 1. 先启动外部依赖（用户已有）
ros2 run <your_imu_package> <imu_node>       # 启动IMU驱动 → /imu/data
ros2 run teleop_twist_joy <...>               # 启动手柄驱动 → /cmd_vel

# 2. 一键启动本项目所有节点
ros2 launch bringup.launch.py

# 3. 带参数启动
ros2 launch bringup.launch.py wheelbase:=0.35 track_width:=0.22

# 4. 验证话题
ros2 topic hz /imu/data          # 应该稳定在 50Hz
ros2 topic hz /odom               # 应该稳定在 50Hz
ros2 topic hz /odometry/filtered  # 应该稳定在 50Hz
ros2 topic echo /odom             # 查看里程计数据
```

## 关键设计决策

| 决策 | 理由 |
|------|------|
| **不开发 IMU 驱动** | 用户已完成，直接订阅 `/imu/data` 即可 |
| **EKF 使用 ROS2 官方包** | `robot_localization` 是成熟的 EKF/UKF 实现，无需重复造轮子 |
| **后轮速度用 Float64 话题** | 避免编译自定义 .msg 文件的复杂度，两个独立话题也便于调试 |
| **50Hz 统一频率** | IMU 读取频率为 50Hz，里程计和 EKF 同步此频率避免插值误差 |
| **看门狗机制** | 手柄断连 500ms 后自动停车，保障安全 |
| **Python 实现** | 阿克曼差速和里程计为轻量算法，Python 性能足够（13ms/周期 < 20ms） |
