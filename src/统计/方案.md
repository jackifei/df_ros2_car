# 基于ROS2 Jazzy的阿克曼小车导航系统方案

## 1. 项目概述

### 1.1 项目背景
本项目开发一套基于ROS2 Jazzy的室内阿克曼结构小车导航与里程计系统。小车采用前轮舵机转向、后轮双电机独立驱动的阿克曼底盘结构，搭载6轴IMU传感器（3轴加速度计 + 3轴陀螺仪），IMU驱动已完成（外部节点，50Hz发布 `/imu/data`），手柄遥控已实现Twist发布。

### 1.2 项目目标
- 实现阿克曼转向的电子差速控制
- 以50Hz频率读取6轴IMU数据，融合IMU与轮式里程计，发布精准的里程计信息
- 利用ROS2 `robot_localization` 包进行多传感器融合定位
- 保留手柄遥操作接口（已发布Twist）并基于Twist计算与发布里程计

### 1.3 硬件平台

| 组件 | 规格 |
|------|------|
| 小车底盘 | 阿克曼结构（前轮转向 + 后轮独立驱动） |
| 前轮转向 | 舵机驱动 |
| 后轮驱动 | L/R 双电机，可独立调速（驱动层已完成） |
| IMU | 6轴：3轴加速度计 + 3轴陀螺仪（**驱动已完成，外部节点**） |
| IMU读取频率 | **50 Hz**（软件设定，Δt = 20 ms），发布 `/imu/data` |
| 手柄 | 已实现Twist发布 |
| 导航环境 | 室内 |

### 1.4 软件平台

| 组件 | 版本/说明 |
|------|-----------|
| 操作系统 | Ubuntu 24.04 |
| ROS2 | Jazzy Jalisco |
| 构建工具 | colcon |
| 语言 | Python 3 |

## 3. 模块设计

### 3.1 阿克曼电子差速模块

**实现文件**: `src/joystick_bridge/ackermann_differential.py`

#### 数学模型

阿克曼转向几何中，车辆转弯时四个轮子绕同一个瞬心旋转。

**参数定义**：
- **L** = 轴距（前后轴间距）
- **W** = 后轮轮距（左右后轮间距）
- **δ** = 前轮转向角（舵机角度）
- **v** = 车辆质心线速度
- **ω** = 车辆角速度
- **R** = 质心转弯半径

**核心公式**：

```
R = L / tan(δ)                              (转弯半径)
v_left  = v × (R - W/2) / R                 (内轮速度)
v_right = v × (R + W/2) / R                 (外轮速度)

// 等效转向角反算（手柄Twist → δ）：
δ = atan(L × ω / v)         当 v ≠ 0
δ = ω                       当 v ≈ 0  (原地旋转)
```

#### 处理逻辑
1. 接收 `/cmd_vel` 的线速度 `v` 和角速度 `ω`
2. 反算等效前轮转向角 `δ`
3. 根据阿克曼模型计算左右后轮差速
4. 发布 `/wheel_speeds`，同时送入里程计

### 3.2 IMU模块（6轴，50Hz，外部已有）

> **IMU 驱动已完成**，发布 `/imu/data` (sensor_msgs/Imu) @50Hz。
> 本项目直接订阅该话题，无需重复开发驱动。
> robot_localization 的 EKF 节点会自动订阅 `/imu/data` 进行融合。

#### 传感器规格

| 参数 | 值 |
|------|-----|
| 传感器类型 | 6轴 IMU |
| 数据内容 | 3轴加速度计（ax, ay, az）+ 3轴陀螺仪（gx, gy, gz） |
| 软件读取频率 | **50 Hz** |
| 采样周期 Δt | **20 ms（0.02 s）** |
| 加速度量程 | ±16 g |
| 陀螺仪量程 | ±2000 °/s |

#### 处理流程

```
50Hz定时器触发
    │
    ▼
I²C/SPI读取12字节原始数据 (6轴×2字节)
    │
    ▼
物理量转换 (LSB → m/s², rad/s)
    │
    ▼
零偏补偿 (启动时静止校准200帧=4秒)
    │
    ▼
低通滤波 (截止频率25Hz, α≈0.15)
    │
    ▼
四元数姿态更新 (RK1积分, dt=0.02s)
    │
    ▼
重力补偿 (线性加速度 = 原始加速度 - Rᵀ×g)
    │
    ▼
协方差填充
    │
    ▼
发布 /imu/data @50Hz
```

#### 关键算法

| 算法 | 说明 |
|------|------|
| 零偏校准 | 上电后采集200帧静止数据取均值（4秒@50Hz） |
| 低通滤波 | 一阶IIR: `y[n]=α·x[n]+(1-α)·y[n-1]`，α=2π·fc·Δt≈0.15 |
| 姿态更新 | 四元数一阶龙格-库塔法 (RK1) 积分陀螺仪角速度 |
| 重力补偿 | 通过旋转矩阵将重力矢量变换到机体坐标系后减去 |

### 3.3 里程计计算模块

**实现文件**: `src/joystick_bridge/wheel_odometry.py`

#### 运动学模型

```
线速度 v   = (v_left + v_right) / 2
角速度 ω   = (v_right - v_left) / W

位置增量 (Δt = 0.02s):
dx = v × cos(θ) × 0.02
dy = v × sin(θ) × 0.02
dθ = ω × 0.02
```

#### 里程计消息规范

| 字段 | 值 | 说明 |
|------|-----|------|
| `header.frame_id` | `"odom"` | 里程计坐标系 |
| `child_frame_id` | `"base_link"` | 机器人本体坐标系 |
| `pose.pose.position` | (x, y, 0) | 2D平面位置 |
| `pose.pose.orientation` | 偏航角四元数 | 仅yaw非零 |
| `twist.twist.linear` | (v, 0, 0) | 前进方向线速度 |
| `twist.twist.angular` | (0, 0, ω) | 绕Z轴角速度 |
| 协方差矩阵 | 对角阵 | x/y方差0.01, yaw方差0.02 |

### 3.4 robot_localization EKF融合

**配置文件**: `src/robot_localization_config/config/ekf_params.yaml`

#### 融合架构

```
传感器输入 (均为50Hz):
- /odom:         轮式里程计 → 线速度vx + 角速度vyaw
- /imu/data:     6轴IMU → 角速度vyaw + 加速度ax,ay + 偏航角yaw

EKF状态 (15维):
[x, y, z, roll, pitch, yaw, vx, vy, vz, vroll, vpitch, vyaw, ax, ay, az]

EKF输出 (50Hz):
- /odometry/filtered: 融合里程计
- /tf:                odom → base_link
```

#### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `frequency` | 50.0 | 与IMU读取频率一致 |
| `sensor_timeout` | 0.06 | 60ms，容忍3个IMU周期丢帧 |
| `two_d_mode` | true | 2D室内平面导航 |
| `world_frame` | odom | 世界坐标系 |
| `odom0_config` | vx + vyaw | 轮式里程计提供速度观测 |
| `imu0_config` | yaw + vyaw + ax + ay | IMU提供姿态+角速度+加速度 |
| `imu0_remove_gravitational_acceleration` | true | 自动去除重力 |

### 3.5 手柄接口模块

**实现文件**: `src/joystick_bridge/joystick_bridge_node.py`

#### 责任
1. 订阅 `/cmd_vel` (geometry_msgs/msg/Twist)
2. 解析 `linear.x`（线速度）和 `angular.z`（角速度）
3. 由线速度和角速度反算等效前轮转向角
4. 调用阿克曼电子差速模块计算L/R轮速
5. 发布自定义消息 `/wheel_speeds`
6. 驱动轮式里程计更新，发布 `/odom` + `/tf`

## 4. 50Hz系统时序分析

### 4.1 主循环耗时预算

```
┌────────────────── 50Hz 主循环 (20ms周期) ──────────────────┐
│                                                             │
│  I²C读取IMU          ~1ms    ██                             │
│  数据转换+滤波        ~2ms    ████                           │
│  姿态解算             ~3ms    ██████                         │
│  EKF预测+更新         ~5ms    ██████████                     │
│  发布Odom+TF          ~2ms    ████                           │
│  IDLE                 ~7ms    ██████████████ (35%裕量)       │
│                                                             │
│  总计算耗时: ~13ms / 周期: 20ms / CPU利用率: ~65%            │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 时序约束

| 参数 | 值 | 说明 |
|------|-----|------|
| IMU读取周期 | 20 ms (50 Hz) | 固定周期，系统时钟基准 |
| 单次I²C读取耗时 | ~1 ms | 12字节数据（6轴×2字节） |
| 姿态解算耗时 | ~3 ms | 四元数积分+滤波 |
| EKF更新耗时 | ~5 ms | 15状态EKF预测+更新 |
| 余量 | ~7 ms | 35% CPU空闲 |
| 传感器超时容忍 | 60 ms | 3个IMU周期，允许偶发丢帧 |

## 9. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| IMU零偏漂移（温漂） | 偏航角发散 | 上电4秒静止校准；EKF融合轮式里程计抑制 |
| IMU 50Hz丢帧 | EKF观测缺失 | sensor_timeout=60ms，容忍1-2帧丢失 |
| 50Hz循环超时 | 发布延迟抖动 | 控制处理<15ms，35%时序裕量 |
| 轮子打滑 | 轮式里程计漂移 | IMU加速度补偿；EKF自动降权 |
| 阿克曼参数不准 | 转弯内外轮速比错误 | 实测轴距/轮距；实车转弯标定 |
| EKF发散 | 定位完全失效 | 仿真验证→实车渐进调参→监控协方差 |
| 时间戳未对齐 | 融合误差增大 | 统一时钟源；50Hz同步触发 |
| 手柄延迟 | 遥控响应不连续 | 保持上一帧值；IMU冗余补充 |

## 附录A：阿克曼转向几何

```
                    瞬心 (转向中心)
                       ●
                      /|\
                     / | \
                    /  |  \
                   /   |   \
                  /    R    \
                 /     |     \
                /      |      \
   前轮(左) →  ●-------|-------● ← 前轮(右)
               |\      |      /|
               | \     |     / |
               |  \    L    /  |
               |   \   |   /   |
               |    \  |  /    |
               |     \ | /     |
               |      \|/      |
   后轮(左) →  ●-------●-------● ← 后轮(右)
               |<- W/2 →|← W/2 →|
               |<------ W ----->|

   其中:  R = L / tan(δ)  (转弯半径)
         v_left  = ω × (R - W/2)
         v_right = ω × (R + W/2)
```
