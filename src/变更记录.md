# 变更记录 (Changelog)

> ROS2 Jazzy — rosrobot 小车项目  
> 控制方案: 同步后驱 + 前轮转向 (Ackermann-style)  
> 仿真平台: Gazebo Harmonic (`gz sim`)  
> 日期: 2026-06-19 ~ 2026-06-20

## 第一阶段：创建 rosrobot_bringup_two 包

**目标**: 将差速驱动替换为同步后驱 + 前轮转向方案，转向和驱动独立受控（类似汽车油门/方向盘模型）。

### 新建文件

| 文件 | 说明 |
|------|------|
| `rosrobot_bringup_two/package.xml` | 包清单，依赖 `rosrobot_description`、`robot_state_publisher`、`rviz2` 等 |
| `rosrobot_bringup_two/CMakeLists.txt` | 构建配置，安装 launch/config/scripts 到 share/lib |
| `rosrobot_bringup_two/launch/display_robot.launch.py` | RViz 显示 + 控制启动文件（keyboard/joystick 模式） |
| `rosrobot_bringup_two/scripts/cmd_vel_to_joints_sync.py` | **核心**: `/cmd_vel` → `/joint_states` 转换，同步后驱逻辑 |
| `rosrobot_bringup_two/scripts/publish_robot_description.py` | 将 URDF 以 Transient Local QoS 发布到 `/robot_description` |
| `rosrobot_bringup_two/config/rosrobot.rviz` | RViz 布局：Grid + TF + RobotModel |

### 核心变更：cmd_vel_to_joints_sync.py

**与差速方案 (`rosrobot_bringup/scripts/cmd_vel_to_joints.py`) 的区别：**

| 维度 | 差速方案 (旧) | 同步后驱 (新) |
|------|-------------|--------------|
| 后轮速度 | `left_vel = v/r - ω·sep/r`<br>`right_vel = v/r + ω·sep/r` | `wheel_vel = v/r`（两轮同值） |
| 转弯方式 | 转向角 + 左右轮速差 同时作用 | **仅前轮转向角** |
| 参数 | `wheel_radius`, `wheel_separation`, `max_steering_angle` | `wheel_radius`, `max_steering_angle`, `steering_scale` |
| 控制模型 | 差速机器人 | 汽车式 Ackermann |

**控制逻辑 (伪代码):**
```
linear.x  → 后轮转速 = linear.x / wheel_radius     (油门，独立)
angular.z → 转向角   = clamp(angular.z, ±0.7)      (方向盘，独立)
两通道完全独立，可同时非零 → 驱动中转弯
```

## 第三阶段：实现 Gazebo + SLAM 建图（Phase ①~⑧）

### Phase ① — URDF 添加仿真标签

**文件**: `rosrobot_description/urdf/rosrobot.urdf`

添加内容：
- `<ros2_control>` 标签：声明 4 个关节的硬件接口（后轮 velocity、前轮 position）
- `<gazebo>` LiDAR 传感器标签：360° 激光雷达，15 Hz
- `<gazebo>` 后轮摩擦系数标签

```xml
<ros2_control name="GazeboSimSystem" type="system">
  <hardware>
    <plugin>gz_ros2_control/GazeboSimSystem</plugin>  <!-- 最初为 gazebo_ros2_control/GazeboSimSystem -->
  </hardware>
  <joint name="lh_joint"><command_interface name="velocity"/>...</joint>
  <joint name="rh_joint"><command_interface name="velocity"/>...</joint>
  <joint name="lq_joint"><command_interface name="position"/>...</joint>
  <joint name="rq_joint"><command_interface name="position"/>...</joint>
</ros2_control>

<gazebo>
  <plugin name="gz_ros2_control::GazeboSimROS2ControlPlugin"
          filename="libgz_ros2_control-system.so">
    <parameters>$(find rosrobot_description)/config/ros2_control.yaml</parameters>
  </plugin>
</gazebo>
```

### Phase ② — ros2_control 控制器配置

**文件**: `rosrobot_description/config/ros2_control.yaml` (新建)

3 个控制器：
- `joint_state_broadcaster` — 只读，从 Gazebo 读取关节状态 → `/joint_states`
- `rear_wheel_controller` — `velocity_controllers/JointGroupVelocityController`，控制 `lh_joint` + `rh_joint`
- `steering_controller` — `position_controllers/JointGroupPositionController`，控制 `lq_joint` + `rq_joint`

### Phase ③ — rosrobot_gazebo 仿真包

**新建包**: `rosrobot_gazebo/`

| 文件 | 说明 |
|------|------|

| `worlds/room.world` | 10×10m 封闭房间，含 2 个障碍物（圆柱 + 方块），SDF 1.9 |

### Phase ④ — cmd_vel_gazebo_bridge.py

**文件**: `rosrobot_bringup_two/scripts/cmd_vel_gazebo_bridge.py` (新建)

**功能**: `/cmd_vel` (Twist) → 两个控制器命令 (Float64MultiArray)

```python
# 后轮速度: linear.x → 车轮角速度 (两轮同值)
wheel_angular_vel = linear_x / wheel_radius
rear_msg.data = [wheel_angular_vel, wheel_angular_vel]

# 前轮转向: angular.z → 转向角 (两轮同值, clamped)
target_angle = clamp(angular_z * steering_scale, -max, +max)
steer_msg.data = [target_angle, target_angle]
```

### Phase ⑤ — ackermann_odometry.py

**文件**: `rosrobot_bringup_two/scripts/ackermann_odometry.py` (新建)

**功能**: `/joint_states` → `/odom` + `odom→base_link` TF（自行车模型）

```
模型:  v = ω_wheel × r
       ω_body = v × tan(δ) / L
       位姿积分: Δx = v × cos(θ+Δθ/2) × Δt
                Δy = v × sin(θ+Δθ/2) × Δt
                Δθ = ω_body × Δt
参数:  wheel_radius=0.05m, wheelbase=0.306m
```

## 第四阶段：Gazebo Classic → Gazebo Harmonic 迁移

**原因**: Ubuntu 24.04 + ROS2 Jazzy 官方不支持 Gazebo Classic (`gazebo` 命令不提供)。  
ROS2 Jazzy 推荐的仿真器是 Gazebo Harmonic (`gz sim`)。

### 迁移清单

| 组件 | Classic (旧) | Harmonic (新) |
|------|-------------|--------------|
| 启动命令 | `gazebo --verbose` | `gz sim` (via `ros_gz_sim/gz_sim.launch.py`) |
| 生成模型 | `spawn_entity.py -topic` | `ros_gz_sim create -topic` |
| 系统插件 | 无 | `gz_ros2_control::GazeboSimROS2ControlPlugin` |
| 硬件插件 | `gazebo_ros2_control/GazeboSimSystem` | `gz_ros2_control/GazeboSimSystem` |
| LiDAR 桥接 | `libgazebo_ros_ray_sensor.so` (内嵌) | `ros_gz_bridge parameter_bridge` (外部) |
| LiDAR 类型 | `ray` | `gpu_lidar` |
| SDF 版本 | 1.6 | 1.9 |
| 环境变量 | 无 | `GZ_SIM_SYSTEM_PLUGIN_PATH=/opt/ros/jazzy/lib/` |
| 包依赖 | `gazebo_ros`, `gazebo_ros2_control` | `ros_gz_sim`, `gz_ros2_control`, `ros_gz_bridge` |
| URDF 宏 | 无 | `$(find ...)` 在 `load_urdf()` 中手动展开 |

### 迁移修改的文件

| 文件 | 修改内容 |
|------|---------|
| `rosrobot_description/urdf/rosrobot.urdf` | 硬件插件名、LiDAR 类型、新增 system plugin 标签 |
| `rosrobot_description/config/ros2_control.yaml` | 新增 `use_sim_time: true` |
| `rosrobot_description/package.xml` | 依赖名更新 |
| `rosrobot_gazebo/worlds/room.world` | SDF 版本 1.6 → 1.9 |
| `rosrobot_gazebo/launch/spawn_robot.launch.py` | 完全重写：`gz sim` + `ros_gz_sim create` + bridge |
| `rosrobot_gazebo/package.xml` | 依赖名更新 |
| `rosrobot_bringup_two/launch/gazebo_bringup.launch.py` | 重写：使用新 spawn launch |
| `rosrobot_bringup_two/launch/slam_bringup.launch.py` | 重写：使用新 spawn launch + slam_toolbox |
| `rosrobot_bringup_two/launch/display_robot.launch.py` | `load_urdf()` 支持 `$(find ...)` 展开 |

## 编译与运行

```bash
# 安装依赖 (Ubuntu 24.04)
sudo apt install ros-jazzy-ros-gz-sim \
                 ros-jazzy-gz-ros2-control \
                 ros-jazzy-ros-gz-bridge \
                 ros-jazzy-ros2-control \
                 ros-jazzy-ros2-controllers \
                 ros-jazzy-slam-toolbox

# 编译
cd ~/df_pj/df_ackermann
colcon build --packages-select rosrobot_description rosrobot_gazebo rosrobot_bringup_two
source install/setup.bash

# 运行模式 1: RViz 显示 + 键盘控制
ros2 launch rosrobot_bringup_two display_robot.launch.py
ros2 run teleop_twist_keyboard teleop_twist_keyboard

# 运行模式 2: Gazebo Harmonic 仿真
ros2 launch rosrobot_bringup_two gazebo_bringup.launch.py
ros2 run teleop_twist_keyboard teleop_twist_keyboard

# 运行模式 3: Gazebo + SLAM 在线建图
ros2 launch rosrobot_bringup_two slam_bringup.launch.py
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

## 已知注意事项

1. **`GZ_SIM_SYSTEM_PLUGIN_PATH`**: 如果 Gazebo 找不到 `gz_ros2_control` 插件，执行：
   ```bash
   export GZ_SIM_SYSTEM_PLUGIN_PATH=/opt/ros/jazzy/lib/
   ```

2. **里程计 (`ackermann_odometry.py`)**: 依赖 `/joint_states` 话题的正常发布。如果 Gazebo 控制器未成功加载，里程计不会产生数据，SLAM 无法建图。

3. **`ros_gz_bridge` 话题路径**: LiDAR 数据在 Gazebo 内的完整路径为 `/model/rosrobot/link/lidar_Link/sensor/lidar/scan`，bridge 自动 remap 到 `/scan`。如果模型名或传感器名改变，需要同步修改 bridge 参数。

4. **`use_sim_time`**: 仿真模式下所有节点必须设置为 `true`，否则 TF 时间戳不匹配会导致数据丢失。
