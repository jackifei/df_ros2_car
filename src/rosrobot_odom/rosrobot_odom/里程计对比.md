| 文件路径 | [rosrobot_odom/rosrobot_odom/wheel_odometry.py](rosrobot_odom/rosrobot_odom/wheel_odometry.py) | [rosrobot_odom/rosrobot_odom/wheel_odom.py](rosrobot_odom/rosrobot_odom/wheel_odom.py) |
| 类名 | `WheelOdometry` | `WheelOdomFusion` |

## 一、运动学模型与传感器使用

这是最核心的差异。

### `wheel_odometry.py` — 单源角速度

```
线速度 v = (v_left + v_right) / 2    ← 后轮编码器
角速度 ω = ω_imu                     ← IMU 陀螺仪（唯一来源）
```

- **仅使用 2 种传感器**：后轮编码器 + IMU 陀螺仪
- 角速度**完全依赖 IMU**，IMU 漂移会导致航向角持续偏移
- 前轮转向角传感器数据**未使用**
- 轴距 L 和轮距 W **仅用于日志**，不参与运动学计算

### `wheel_odom.py` — 多源融合

```
线速度 v = 编码器测量                    ← 后轮编码器
角速度 ω = 互补滤波(                    ← 双源融合
              ω_imu,                   ← IMU 陀螺仪（高频响应）
              ω_steer = v·tan(δ)/L     ← 转向角模型（无漂移）
          )
航向角 ← IMU yaw 缓慢修正               ← 绝对参考约束
```

- **使用全部 4 种传感器**：后轮编码器 + 前轮转向角 + IMU 陀螺仪 + IMU 航向角
- 角速度双源互补：IMU 提供高频响应，转向角模型修正漂移
- IMU 航向角作为**绝对参考**，缓慢拉回积分漂移
- 轴距 L **实际参与计算**（自行车模型）

### 传感器分工

```
         ┌──────────────────────────────────────────┐
         │           wheel_odom.py 传感器分工         │
         │                                           │
         │  后轮编码器  ──→  线速度 v  (唯一来源)      │
         │  前轮转向角  ──→  角速度ω  (漂移修正)       │
         │  IMU 陀螺仪  ──→  角速度ω  (高频主导)       │
         │  IMU 航向角  ──→  航向θ  (绝对约束)         │
         │                                           │
         └──────────────────────────────────────────┘
```

## 二、数值积分方法

| 维度 | `wheel_odometry.py` | `wheel_odom.py` |
|------|---------------------|-----------------|
| 积分方法 | 前向欧拉法 | **中点法（RK2）** |
| 精度阶数 | 一阶 | **二阶** |
| 转弯轨迹 | 向外偏移 | 更接近真实圆弧 |

### 前向欧拉法（旧）

```python
# 用当前角度计算位移 → 转弯时系统性地外偏
cos_theta = cos(self.theta)
sin_theta = sin(self.theta)
self.x += v * cos_theta * dt
self.y += v * sin_theta * dt
self.theta += omega * dt
```

### 中点法（新）

```python
# 用 Δt 中点时刻的角度 → 更接近真实圆弧轨迹
delta_theta = omega * dt
theta_mid = self.theta + delta_theta / 2.0

self.x += v * cos(theta_mid) * dt
self.y += v * sin(theta_mid) * dt
self.theta += delta_theta
```

**对小车的影响**：直线行驶几乎无差异；转弯时（尤其 30° 以上转角），中点法轨迹更准确。对于有 SLAM 需求的场景，这个差异会体现在地图一致性上。

## 三、互补滤波器（核心新增特性）

`wheel_odom.py` 的核心创新是**互补滤波器融合角速度**。

### 工作原理

```
ω_fused = α_dyn · ω_imu + (1 - α_dyn) · ω_steer
```

其中 `α_dyn` 是**动态权重**，根据车速相对于 `max_speed` 的比例调整：

```
speed_factor = min(1.0, |v| / max_speed)
α_dyn = α_gyro × (0.3 + 0.7 × speed_factor)
```

以 `max_speed=0.3 m/s`、`α_gyro=0.92` 为例：

| 车速 | speed_factor | α_dyn | 信任偏向 |
|------|-------------|-------|----------|
| 静止 ~ 0.02 m/s | 0.07 | ~0.32 | 偏转向模型（无漂移） |
| 0.1 m/s | 0.33 | ~0.49 | 转向模型略占优 |
| 0.2 m/s | 0.67 | ~0.71 | IMU 略占优 |
| ≥0.3 m/s | 1.0 | ~0.92 | IMU 主导（响应快） |

### 为什么这样设计

- **低速时** IMU 陀螺仪信噪比低（有用信号小），且零偏漂移相对显著，而转向模型 `v·tan(δ)/L` 不受漂移影响
- **高速时** IMU 陀螺仪动态响应好，转向模型受机械间隙、轮胎侧滑影响大
- **直行时**（ω_steer≈0）直接退化为纯 IMU，转向模型不参与

### 低通滤波

融合结果经过一阶低通滤波（fc≈20Hz），去除高频噪声同时保持 50Hz 响应能力。

## 四、IMU 航向角修正（新增特性）

`wheel_odom.py` 使用 IMU 提供的绝对航向角（yaw）作为**长期约束**。

### 初始化（v1.1 修复）

```
收集前 30 帧，每帧记录 delta = theta - yaw_imu
计算 offset = mean(delta_0 ... delta_29)
```

**关键设计**：记录的是 `theta - yaw_imu` 差值而非 yaw_imu 绝对值。即使小车在运动中初始化，`theta` 和 `yaw_imu` 同步变化，差值序列保持恒定，均值等于真实的初始偏移。旧版直接平均 yaw_imu 绝对值在运动中会算出错误偏移。

### 运行时修正

```
error = (yaw_imu + offset) - theta      # 归一化到 [-π, π]
correction = error · gain · dt          # 缓慢修正
theta += correction
```

### 增益调度

| 航向误差 | 增益倍数 | 策略 |
|----------|----------|------|
| < 2° | ×1 | 正常缓慢修正 |
| 2° ~ 5° | ×2 | 加速修正 |
| > 5° | ×4 | 快速拉回（可能发生了滑移或碰撞） |

增益基础值默认 0.005，意味着即使持续 5° 偏差，也需要约数十秒才能完全修正——**足够慢以避免磁干扰抖动**，但**足够快以消除陀螺仪累积漂移**。

## 五、零速检测（新增特性）

`wheel_odom.py` 包含静止检测逻辑，`wheel_odometry.py` 没有。

```python
# 连续 N 帧同时满足：
#   |v| < 0.005 m/s   → 轮子未转动
#   |ω_imu| < 0.01 rad/s → 车身未旋转
# → 判定为静止，发布当前位姿但不积分
```

**作用**：
- 防止编码器微小噪声在静止时持续积分导致位置漂移
- 为 SLAM 提供真实的零速约束
- 静止时仍发布 odom（保持数据流不中断）

## 六、异常保护

| 保护机制 | `wheel_odometry.py` | `wheel_odom.py` |
|----------|---------------------|-----------------|
| 输入有效性检查 | ✅ v, omega | ✅ v, δ, ω_imu, yaw_imu **全部** |
| dt ≤ 0 保护 | ✅ | ✅ |
| dt 跳变保护 | ❌ | ✅ >0.2s 时重置积分 |
| 速度限幅 | ❌ | ✅ ±max_speed×1.5 |
| 角速度限幅 | ❌ | ✅ ±5 rad/s |
| 转向角限幅 | N/A | ✅ ±60° |
| tan 极值保护 | N/A | ✅ 上限 ±50 |
| IMU 陀螺仪死区 | N/A | ✅ \|ω\|<0.005 rad/s 时置零 |
| 零速检测 | ❌ | ✅ 连续10帧判定 |
| 限流告警 | ❌ | ✅ 前5次+每500次 |
| yaw 运动初始化 | N/A | ✅ 记录差值（运动中也可正确初始化） |
| /path 时间戳一致性 | N/A | ✅ 与 odom/TF 使用同一时间戳 |
| 参数合法性校验 | ❌ | ✅ max_speed>0, wheelbase>0 |

## 七、协方差矩阵

| 协方差项 | `wheel_odometry.py` | `wheel_odom.py` |
|----------|---------------------|-----------------|
| pose x/y | 固定 0.01 | **动态** 0.002 + 0.01×(v/max_speed) |
| pose yaw | 固定 0.05 | **动态** 0.002 + 0.02×\|ω\| |
| twist vx | 固定 0.01 | **动态** 0.002 + 0.005×(v/max_speed) |
| twist vyaw | 固定 0.03 | **动态** 0.002 + 0.01×\|ω\| |

**动态协方差的意义**：以 `max_speed` 为基准归一化速度，适配不同车速的小车。下游节点（如 `robot_localization` EKF）可根据协方差正确加权。

## 八、接口对比

### `wheel_odometry.py`

```python
odom = WheelOdometry(node, wheelbase=0.36, track_width=0.39)

# 旧接口（2 个传感器）
odom.update_with_imu(v=0.5, omega_imu=0.1, timestamp=now)

# 备用接口
odom.update_from_twist(linear_vel=0.5, angular_vel=0.1, timestamp=now)
```

### `wheel_odom.py`（v1.1）

```python
odom = WheelOdomFusion(node, wheelbase=0.306, track_width=0.39,
                       wheel_radius=0.062,
                       max_speed=0.3,              # ← 新增：小车最大速度
                       alpha_gyro=0.92,            # 互补系数
                       yaw_correction_gain=0.005)  # 航向修正增益

# 新接口（4 个传感器）
odom.update(
    v=0.15,             # 编码器线速度 m/s
    steering_angle=0.1, # 前轮转向角 rad ← 新增
    omega_imu=0.05,     # IMU z轴角速度 rad/s
    yaw_imu=1.57,       # IMU 航向角 rad ← 新增（可选）
    timestamp=now
)
```

### 兼容性说明

`wheel_odom.py` 的 `update()` 的 `yaw_imu` 参数为可选（默认 `None`）。不提供时，航向角修正功能自动禁用，里程计仍可正常运行（退化为互补滤波模式）。

## 九、参数调优建议

### `max_speed`（小车最大速度，新增）

| 值 | 适用场景 |
|----|----------|
| **0.3** | **你的小车（默认值）** |
| 0.5 ~ 1.0 | 中速小车 |
| > 1.0 | 高速小车 |

影响：速度限幅上限（×1.5）、互补滤波速度归一化、协方差缩放基准。

### `alpha_gyro`（互补滤波系数）

| 值 | 行为 | 适用场景 |
|----|------|----------|
| 0.85 | 偏信任转向模型 | 转向角传感器精度高、IMU 漂移大 |
| **0.92** | **推荐值** | 通用场景 |
| 0.98 | 偏信任 IMU | IMU 质量好（如 BNO055、ICM-20948） |

### `yaw_correction_gain`（航向修正增益）

| 值 | 行为 | 适用场景 |
|----|------|----------|
| 0.001 | 极慢修正 | IMU yaw 波动大（低成本磁力计） |
| **0.005** | **推荐值** | 通用场景 |
| 0.02 | 较快修正 | IMU yaw 稳定（如带硬件融合的 IMU） |

### 测试方法

1. 让小车走一个 2m×2m 的正方形，回到起点看闭合误差
2. 原地旋转 5 圈，看航向角漂移
3. 在 RViz2 中观察 `/path` 轨迹线是否平滑

## 十、降级策略

如果运行中某个传感器故障，`wheel_odom.py` 可以继续工作：

| 故障传感器 | 影响 | 剩余能力 |
|------------|------|----------|
| 前轮转向角 | 无法计算 ω_steer | 退化为纯 IMU 模式（等价旧版） |
| IMU 陀螺仪 | 无法获取 ω_imu | 退化为纯自行车模型 |
| IMU 航向角 | 无绝对航向约束 | 互补滤波仍工作，但长期有漂移 |
| 后轮编码器 | 无法获取 v | **完全失效**（线速度唯一来源） |

> 注：`update()` 中 ω_imu 为必选参数。如果要优雅降级，可在宿主节点中检测传感器状态，故障时传 0.0。

## 十一、总结

| 维度 | `wheel_odometry.py` | `wheel_odom.py` |
|------|---------------------|-----------------|
| 传感器利用 | 编码器 + IMU 陀螺仪 | 编码器 + 转向角 + IMU 陀螺仪 + IMU 航向角 |
| 角速度策略 | 单源（IMU only） | **互补滤波融合（IMU + 自行车模型）** |
| 长期漂移 | 依赖 IMU 自身精度 | **IMU 航向角约束** |
| 积分精度 | 一阶前向欧拉 | **二阶中点法** |
| 抗噪能力 | 固定协方差 | **动态协方差** |
| 零速处理 | 无 | **静止检测 + 锁定** |
| 异常保护 | 基础 | **多层（dt/限幅/tan/告警）** |
| 可调参数 | 0 | **3 个（max_speed, α_gyro, yaw_gain）** |
| 代码行数 | ~270 | ~455 |
| 适用场景 | 通用、IMU 质量好 | **阿克曼底盘、需要长期稳定** |

## 附录：v1.1 Bug 修复记录

以下 bug 在代码审查中发现并修复：

### Bug 1（中等）：IMU yaw 初始化在运动中产生错误偏移

**问题**：旧版直接对 `yaw_imu` 绝对值求平均来计算 offset。如果小车在初始化阶段（前30帧）有旋转运动，`theta` 和 `mean(yaw)` 的变化量不匹配，导致 offset 包含运动分量。

**修复**：改为每帧记录 `delta = theta - yaw_imu`，对差值序列求均值。因为 `theta` 和 `yaw_imu` 通过刚性连接同步变化，差值在运动中保持恒定，均值正确反映真实偏移。

```python
# 旧（有 bug）
self._yaw_samples.append(yaw_imu)
mean_yaw = sum(self._yaw_samples) / len(self._yaw_samples)
self._yaw_offset = self.theta - mean_yaw  # theta 已变化，但 mean_yaw 跨越整个采样期

# 新（已修复）
delta = self.theta - yaw_imu
self._yaw_offset_samples.append(delta)
self._yaw_offset = sum(samples) / len(samples)  # 差值在运动中也稳定
```

### Bug 2（低）：`steering_angle` 未做 NaN/Inf 检查

**问题**：只对 `v` 和 `omega_imu` 做了 `isfinite` 检查，`steering_angle` 为 NaN 时会通过 `math.tan()` 传播污染。

**修复**：在 Step 1 中增加 `math.isfinite(steering_angle)` 检查，同时对 `yaw_imu` 也做独立校验。

### Bug 3（低）：`/path` 轨迹时间戳不一致

**问题**：`_publish_tf` 中 path pose 的 `stamp` 使用 `self.node.get_clock().now()`，而 odom 和 TF 使用传入的 `timestamp`。两者可能相差数毫秒，在精确时间对齐场景（如 sensor fusion）中造成微小不一致。

**修复**：path pose 的 stamp 改为使用传入的 `timestamp`，与 odom/TF 保持一致。

### Bug 4（中等）：速度归一化硬编码 1.0 m/s

**问题**：互补滤波的 `speed_factor = min(1.0, abs(v) / 1.0)` 硬编码归一化基准为 1.0 m/s。对于 max_speed=0.3 m/s 的小车，即使全速行驶 `speed_factor` 也仅 0.3，导致 `α_dyn` 永远达不到设计的上限，滤波器过度偏向转向模型。

**修复**：新增 `max_speed` 参数，`speed_factor = min(1.0, abs(v) / max_speed)`，使归一化基准与小车实际性能匹配。同时协方差也用 `max_speed` 归一化。

### Bug 5（低）：`_apply_yaw_correction` 后 theta 未归一化

**问题**：yaw 修正直接 `self.theta += correction`，不做归一化。虽然 `update()` 末尾会归一化，但如果 `pose` 属性在两次 update 之间被读取（例如被外部定时器查询），可能返回越界的角度值。

**修复**：在 `_apply_yaw_correction` 内部修正后立即执行 `atan2` 归一化。

## 附录 B：v1.2 Bug 修复记录（第二轮审查）

### Bug 6（中）：`_complementary_filter` 中 `max_speed` 作为除数无零值保护

**问题**：`speed_factor = min(1.0, abs(v) / self.max_speed)`（第410行）对 `max_speed=0` 无保护，触发 ZeroDivisionError。而同文件的 `_update_covariance`（第510行）有 `if self.max_speed > 0 else 0.0` 保护，两处不一致。

**修复**：在 `__init__` 增加 `if max_speed <= 0: raise ValueError(...)` 前置校验（fail-fast 原则），同时保证 `self.L > 0`（防止 `_bicycle_omega` 除零）。此后所有除 `max_speed` 和 `self.L` 的路径都安全。

### Bug 7（低）：直行/极低速时 IMU 陀螺仪底噪被积分

**问题**：当 `omega_steer ≈ 0`（直行或车速 < 0.02 m/s），滤波器直接采用 `raw_omega = omega_imu`。IMU 陀螺仪的零偏噪声（典型值 ~0.005 rad/s）在直行时被持续积分，造成微小航向漂移。

**修复**：新增 `GYRO_NOISE_FLOOR = 0.005` 死区常数。当 `|omega_imu| < GYRO_NOISE_FLOOR` 且转向模型无输出时，强制 `raw_omega = 0.0`。如果 IMU 值超出死区（如小车被人旋转），仍正常跟踪。

### 改进（极低）：魔数 0.1 添加注释

`_apply_yaw_correction` 中 `self.filtered_omega += error * gain * 0.1` 的系数 0.1 原无注释，现说明其作用——使偏置约为直接修正的 1/10，平滑且不过度累积。
